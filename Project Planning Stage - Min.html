Description of Each Data¶

In this project, we are handling two datasets, 'sessions.csv' and 'players.csv' that are both containing some relevant data about the players in PLAICraft mincraft server.
The raw data set 'sessions.csv' is a data set that contains list of individual play sessions by each player, including data about the session. This dataset contains 5 specific columns of data:

  • Hashed email
  • Start time
  • End time
  • Original start time
  • Original end time

The start time and end time are both written in untidy format. However, it is also written in original time format.
For further description of difference between start/end time and original time, the original time is the number of milliseconds since January 1, 1970, 00:00:00 UTC.

On the other hand, the raw data set 'players.csv' contains list of all unique players, including data about each player. This data set contains for specific data of each player:

  • Experience(Pro, Veteran, Amateur, Regular, Beginner)
  • Subscribe (True, False)
  • Hased email
  • Played hours
  • Name
  • Gender
  • Age

Types of each data, can be seen in the table above.

2. Questions¶

In this project, we are selecting one broad question, then formulate a specific question to answer by using list of data above. Therefore, the broad question I chose is, "We would like to know which "kinds" of players are most likely to contribute a large amount of data so that we can target those players in our recruiting efforts." By using this broad question, I have formulated specific question, “Can number of occurrences to the server and played_time predict experience in the dataset, 'players.csv'?” This formulated question will reversly follow the broad question and answer it by using the number of occurrences to the server, which demonstrate the contribution of data to the server, to predict the kind of players, which was answered based their own experiece to the minecraft.
The number of occurrences to the server will be calculated by counting the number of data recorded in session.csv file.

Description¶

To answer my question above, I need to make a new data column that represent the number of occurrences to the server. To make this data, I decided to use the count function to count the number of each hashed email appears in session dataset. Also, I decided to delete the columns other than hashed email in session dataset, since the sum of all play time for the individual users are already recorded in the players dataset. Therefore, I only selected hased email column in session dataset and created new column called count, that counts the number of occurrences to the server for individual users. However, the variables in the question are listed in different dataset. So, I chose to merge two datasets in to single file for further exploration. Both file has same column named 'hashed email', I chose to merge to datasets by hashed email column and if hashed email recorded in players dataset does not exist in session data set, the count column will turns to N/A, since the log file for them does not exist.
By observing the simple plot, it mostly seems like the person with high number of occurrences also has high play time. This information is clearly explaning that the person with high number of occurrences does contribute a large amount of data to the server.

4. Methods and Plan¶

The dataset will be more arranged by deleting unnecessary columns to answer the question. The count, expereience, and played hours will be only selected to answer the question.
To answer the question, "Can number of occurrences to the server and played_time predict experience in the dataset, 'players.csv'?", the method that I choose is a classification model, other than regression model. The question is asked to predict the experience of the user, which is a discrete value, not a continuous. Therefore, the classificiation model will better help to find the expereience of the user.
To answer the broad question, we will choose some specific coordinate of prediction, low, mid, and high, to predict the expereince of that specific point. And this will help to answer the broad question, that prediction made in low point is the expereience that contributes the least amount to the server, and high for high contribution.
To put data into the model, the data will first generalize to avoid the biased prediction. The best-k might be tuned at the beggining, since we cannot see the generalized plot, but might be calculated by using fold-cross validation method. The dataset will be splited to 70% of training and 30% of test to avoid the overtrained model. This split will happen after creating tuned-plot and it's prediction, to compare the difference and the splited data will use for both fold cross and k-nn classification with best k.
Lastly, the result in some specific point will be given by the model, and these results will be use to answer the broad question.
