Data Descrition:¶

The obtained data comprises two csv files: "sessions.csv" recording the times each player joined and left the server, "players.csv", which contains basic demographics on each player - identified by their hashed email. The variable names and types are tabulated below:

Sessions.csv variable descriptions¶

  • hashedEmail - encrypted form an email address
  • (start/end)_time - the time log of the beginning/end of each session in the server
  • original_(start/end)_time - same as above, but preprocessed

Players.csv variable descriptions¶

  • experience - describing a player's experience level with Minecraft (amateur, beginner, veteran, regular, pro)
  • hashedEmail - same as in sessions.csv
  • name - name of the player
  • gender - gender of the player
  • played_hours - number of total hours spent on the server
  • Age - age of the player
  • subscribed - whether or not the player is subscribed to the newsletter

I'm unfamiliar with the 'hashform' of emails and the odd numerical representation of the start and end times from the session dataset. If I wanted to address any information regarding the specific times at which players were active at - such as in question 3 - then I would need to learn how to interpret and modify this data into a usable form.

Another possible issue relating to the sessions.csv is that each observation records individual sessions rather than players. If I were interested in player-specific statistics, I would have to find a way effectively combine the two datasets.

I was most invested in the first broad question presented to us, so I intend to utilise the players.csv data set exclusively.

players_url <- "https://raw.githubusercontent.com/clam171/DSCI-100-Project-Planning-Data/6946339e698182089da5e1cbee4317565e4fc387/players.csv"

select_players <- select(players, -c(name, hashedEmail))

Methods and Plan:¶

To perform an analysis on my question of interest, I'd like to perform a series of multivariable classification predictions. I thought this would be an appropriate way to answer my question, since my responsive variable is categorical and my explanatory variables are numerical.

The key inisital step during this process would be splitting and the data. The split would be would probably be around 70% training, 30% testing. Then I would test to see which predictive model would function best (linear regression, nearest neighbors and linear neares neighbors) based on minimal RMSE and RMSPE values.

I will likely be favouring the non-linear predictive models due to the fact that there isn't a linear relationship between Age and played_hours (as seen in PLOT 1). However, I will exploring other predictive models to see if there are any that account for a similar data distribution - perhaps bell curve models.

In order to be certain on making the right selection, I will maximise the 'training' dataset by doing a cross-validation customarily with 5 folds. The cross validation using the vfold_cv function would essentially scramble the training the data in unique 'folds' creating multiple training sets.
